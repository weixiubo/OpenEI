# 302 低延迟感知实验归档

本目录保存一次端侧感知链路低延迟优化实验。该实验用于验证在资源受限设备上缩短首次有效特征输出时间的可行性，属于历史实验材料，不代表当前主线工程需要整目录覆盖。

## 目录布局

```text
302/
├── config/audio_config.py
├── core/music_analyzer.py
├── dance/robot_controller.py
├── deploy.py
├── MODIFICATIONS.md
├── QUICK_REFERENCE.md
└── README.md
```

## 实验目标

- 缩短实时感知分析的启动等待时间。
- 降低连续任务执行前的特征就绪延迟。
- 保持主线接口不变，避免影响语音交互、技能调度和硬件适配层。

## 主要改动

- 将 `analysis_window` 从 `2.0` 秒调为 `1.2` 秒。
- 降低音频有效信号门槛和连续有效帧要求。
- 在节拍检测中显式使用 `hop_length = 512`。
- 缩短任务执行前的最小感知就绪等待。

## 使用原则

- 该目录只作为参数调优和行为对照材料。
- 不建议把实验文件直接覆盖到当前主线。
- 当前主线已经吸收其中低风险收益点，并保留原有接口。
