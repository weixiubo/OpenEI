# 304 语音供应商替换实验说明

## 实验定位

本目录保存一次语音识别供应商替换实验，用于评估把主线语音识别模块从百度服务切换到阿里云 NLS 的可行性。该内容属于历史实验材料，不代表当前主线已经迁移供应商。

## 修改目标

- 保持 `SpeechRecognizer.recognize(audio_data, sample_rate) -> (bool, str)` 外层接口不变。
- 将上层交互运行时与底层语音供应商解耦。
- 评估 appkey 模式和 token 模式下的认证复杂度。

## 文件说明

- `voice/speech_recognition.py`
  - 尝试使用阿里云 NLS 完成同步识别封装。
  - 保持返回值格式不变。
  - 保留音频长度检查和错误处理。
- `config/api_config.py`
  - 增加阿里云 NLS 配置项。
  - 保留百度配置，便于对比。
- `.env.example`
  - 使用占位符展示供应商所需环境变量。

## 结论

- 供应商替换应只影响语音识别适配层，不应影响意图解析、任务调度和硬件执行层。
- 当前主线仍使用百度语音服务，阿里云实验代码仅作为替换方案参考。
- 如果后续迁移，应补充供应商鉴权测试、失败降级测试和端到端回归测试。
