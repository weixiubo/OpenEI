# OpenEI 发布路线

OpenEI 的长期方向是机器人版 Agent 运行时：让开发者用低成本真实机器人快速验证具身任务、技能编排和硬件适配。

路线围绕四个核心指标展开：低成本、五分钟可运行、技能可扩展、硬件可适配。

## v0.1.0

- 稳定 `openei.events`、`openei.tasks`、`openei.skills`、`openei.adapters`、`openei.runtime` 公共接口。
- 提供统一命令行：`quickstart`、`run`、`skill`、`adapter`。
- 提供模拟器、串口、HTTP、MQTT、ROS 2 可选模板。
- 提供内置动作数据和 `skill.yaml` 技能包样例。
- 提供审计日志、失败恢复和基础安全策略。
- 补齐架构、快速开始、技能开发、适配器开发、ROS 2、审计文档。
- 录制一个模拟器演示和一个真实机器人执行视频。

## v0.2.0

- 增加真实视觉模型提供方。
- 增加本地模型和云模型配置。
- 提供更多机器人适配器模板。
- 增加技能包索引和安装源配置。
- 增加任务执行日志可视化。

## v0.3.0

- 形成技能包生态雏形。
- 支持第三方技能目录。
- 支持硬件适配器模板生成和验证。
- 增加任务执行回放、失败分析和安全约束策略。
- 形成一套面向开发者的教程、短文、动图和真实机器人演示材料。

## 传播节奏

1. 发布“机器人版 OpenClaw”概念短文和动图。
2. 发布五分钟模拟器教程。
3. 发布真实机器人执行视频。
4. 邀请社区贡献技能和硬件适配器。
